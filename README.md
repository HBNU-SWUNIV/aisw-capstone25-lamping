### 개요
- 사용자가 로봇의 주행 경로를 직관적으로 확인하거나 주행 중 개입할 수 없어 안정성과 제어 효율성이 저하되는 문제 발생
- Object Detection을 통해 인식된 물체를 회피 대상으로 설정하여 주행 안정성 향상
- 로봇의 영상을 실시간으로 확인하며 조이스틱 또는 지도 입력으로 목표 위치까지 주행할 수 있는 모바일 앱 개발

## <u>Teamate</u> Project Background
- ### 필요성
  - 자율 주행 로봇은 다양한 환경에서 안정적으로 주행하기 위해 주변 신호를 정확히 인식하고, 장애물을 회피하며 최적의 경로를 찾아야 한다.
  - 특히 실내·복합 환경에서 신호 간섭이나 예기치 못한 장애물이 빈번하게 발생하므로, 로봇이 실시간으로 신호를 추적하고 상황에 맞는 회피 경로를 계산할 수 있는 기술이 필요하다.
- ### 기존 해결책의 문제점
  - 기존 자율 주행 시스템은 주로 단순한 센서 기반 신호 추적 및 사전 정의된 경로 탐색 알고리즘에 의존하기 때문에, 예측 불가능한 환경 변화나 신호 손실 상황에 대응하기 어렵다.
  - 장애물 회피 알고리즘은 정적인 장애물에는 효과적이지만, 동적인 환경이나 신호 간섭이 존재하는 복잡한 공간에서는 경로의 효율성이 떨어진다.
  
## System Design

  - ### System Requirements
    - **OS**: Ubuntu 22.04 LTS  
    - **Framework**: ROS2 Humble  
    - **Hardware**: TurtleBot3 Waffle Pi  
    - **Frontend**: Flutter  
    - **Language**: Python, C++  
    - **Libraries/Packages**: OpenCV, Nav2, Cartographer, rclpy, sensor_msgs  
    - **Network**: Wi-Fi 기반 ROS2 통신
    
## Case Study
  - ### Description
    - ROS2 Humble을 기반으로 한 로봇 제어 시스템과 Flutter 애플리케이션 간의 연동을 통해, 로봇의 주행 상태를 실시간으로 모니터링하고 제어 명령을 전달할 수 있도록 구성하였다
    - 실험은 TurtleBot3 Waffle Pi를 활용하여 다양한 실내 환경(복도, 교실, 실험실 등)에서 수행하였으며, 환경 조명 변화나 장애물 위치 변화에도 안정적인 신호 인식과 경로 탐색이 가능함을 확인하였다.

  - ### 기대 효과
    - 실시간 영상 기반 제어 및 모니터링 시스템을 통해 로봇의 사용자 접근성과 주행 안정성이 향상된다.
    - Save Point 기능을 활용하여 반복 이동이나 특정 지점의 효율적인 관리가 가능하다.
    - 경로 주행 중에도 사용자 조작이 가능하여, 예기치 못한 상황 발생 시 유연한 제어가 가능하다.  
  
## Conclusion
  - ### Flutter로 개발한 모바일 앱을 WebSocket으로 로봇 환경과 연동하여 원격 제어 및 상태 모니터링 지원
  - ### ROS2와 SLAM을 기반으로 Camera, LiDAR가 장착된 TurtleBot이 장애물을 회피하며 최적 경로를 탐색하는 자율주행 시스템 구성
